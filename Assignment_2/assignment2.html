Exercise 1 - Conceptual Questions on Supervised Learning I¶

[4 points]

For each part below, indicate whether we would generally expect the performance of a flexible statistical learning method to be better or worse than an inflexible method. Justify your answer.

1.1. The sample size $n$ is extremely large, and the number of predictors $p$ is small.

In this case, a flexible method will perform better than an inflexible method. It's because with a large amount of data and relatively few predictors, a flexible model can learn complex relationships without as high a risk of overfitting. The large number of observations can balances out the higher variance that comes from model flexibility.

1.2. The number of predictors $p$ is extremely large, and the number of observations $n$ is small.

In this case, a flexible method will perform worse than an inflexible method. It's becuase in high-dimensional settings (large p), flexible models are prone to overfitting, especially when the dataset is small. The large parameter space and limited data often lead to very high variance and poor generalization.

1.3. The relationship between the predictors and response is highly non-linear.

In this case,a flexible method will perform better than an inflexible method. It's because flexible methods can more easily adapt to complex, nonlinear relationships. Inflexible models may underfit if the true relationship is far from linear.

1.4. The variance of the error terms, i.e. $\sigma^2 = Var(\epsilon)$, is extremely high.

In this case, a flexible method will perform worse than an inflexible. It's because if there is a great deal of irreducible noise in the data (large $\sigma ^2$), adding extra flexibility often means fitting noise rather than true structure, leading to higher variance of the predictions and worse performance on new data.

# Skeleton code for part (a) to write your own kNN classifier
import numpy as np

class Knn:
# k-Nearest Neighbor class object for classification training and testing
    def __init__(self):
        self.X_train = None
        self.y_train = None
        
    def fit(self, X, y):
        # Save the training data to properties of this class
        self.X_train = X
        self.y_train = y

    def predict(self, X_test, k):
        y_hat = [] # Variable to store the estimated class label for 
        # Calculate the distance from each vector in x to the training data
        for x in X_test:
            distances = np.sqrt(np.sum((self.X_train-x)**2, axis = 1))
            nn_idx = np.argsort(distances)[:k]
            neighbors = self.y_train[nn_idx]
            counts = {}
            for label in neighbors:
                counts[label] = counts.get(label,0)  + 1
            pred_label = max(counts, key=counts.get)
            y_hat.append(pred_label)
        # Return the estimated targets
        return y_hat

# Metric of overall classification accuracy
#  (a more general function, sklearn.metrics.accuracy_score, is also available)
def accuracy(y,y_hat):
    nvalues = len(y)
    accuracy = sum(y == y_hat) / nvalues
    return accuracy

4.5. Some supervised learning algorithms are more computationally intensive during training than testing. What are the drawbacks of the prediction process being slow? In what cases in practice might slow testing (inference) be more problematic than slow training?

  1. What are the drawbacks of the prediction process being slow?
  • There might be real-time delays. If the applicaiton requires real time response, a slow inference impacts on the user's experience (conversational requests, etc.) and leads to mistakes (distance operations, etc.).
  • There might be calability constraints. If the application requires a large number of predictions, a slow inference step becomes a bottleneck. More equipment, labors, and other resources are needed to handle the load in a timely manner.
  • There might be higher operational costs. Even if the resources can be allocated more toward computation, it increases operational costs. This will lead to applications economically unfeasible.
  1. In what cases in practice might slow testing (inference) be more problematic than slow training?
  • High-frequency or streaming applications like self-driving vehicles, distance operations and surgeores, finance tradings require timely response, or there will be catastrophic hazards.
  • Search engines. Serving relevant results to millions of queries per second. If inference is slow, user queries back up quickly, degrading performance and user experience.
  • Smart home devices like voice assistants must respond quickly to user commands; slow inference leads to poor user experience or unresponsiveness.
  • Posts and feeds on social media are ranked in real time for millions of users. If each inference takes too long, it’s a major bottleneck.
  • Online fraud detection: Credit card or e-commerce transactions must be classified (fraud or not) quickly and at large scale. Delays frustrate customers or fail to prevent fraud in time.
